# WF2 사주 입력 + 투자 성향 결과 플로우 구현

## 1. User Story / Motivation

온보딩을 마친 사용자가 **6단계 사주 입력 → 분석 로더 → 투자 성향 결과(Aha 모먼트) → 가입 유도 → 친구 초대**까지 한 흐름으로 이어져야 한다.

- 사주 입력은 한 화면에 한 가지 정보만 묻는 6단계로 나눠 **인지 부하를 최소화**한다. (성별 → 이름 → 생년월일 → 태어난 시간 → 출생지 → 진태양시 보정)
- 태어난 시간을 모르는 사용자는 "시주 제외, 3주 기반 분석" 폴백으로 이탈 없이 완주할 수 있어야 한다.
- 출생지·경도·균시차를 바탕으로 진태양시를 보정하여 **신뢰도 시그널**(계산 결과 프리뷰)을 제공한다.
- 분석 결과는 **단일 결과 화면**(Hero 유형 + 사주 원국 미니 + 오행 균형 + 강점/주의점/접근 참고)으로 아하 모먼트를 만들고, 스크롤 하단에서 PRO 페이월 및 가입 CTA로 전환을 유도한다.
- 결과 직후 "결과 저장하기" 가입 유도 화면과 "친구 초대 + 공유 카드" 화면으로 MVP 바이럴 루프를 완성한다.

운테크의 포지셔닝("사주로 내 투자 성향과 오늘의 리스크를 점검")상 이 플로우가 **첫 사용 경험의 핵심**이며, 종목 추천이 아닌 성향·리스크 점검 관점에서 메시지를 구성한다.

## 2. Functional Requirements

### 2.1 공통 — 스텝 진행 바 & 네비게이션
- FR-C1. 6단계 입력 화면(Step 1~6) 상단에는 진행 바가 표시되며, 현재 스텝에 따라 1/6 → 6/6까지 채워진다.
- FR-C2. 각 스텝 상단에 "뒤로" 버튼을 제공한다. Step 1에서 뒤로 누르면 온보딩 종료 지점(WF1 마지막 화면 또는 홈)으로 돌아간다.
- FR-C3. 각 스텝 하단에는 CTA 버튼이 고정되며, Step 1~5는 "다음", Step 6은 "사주 분석 시작" 라벨을 표시한다.
- FR-C4. CTA는 해당 스텝의 필수 입력이 완료되기 전까지 비활성화된다(아래 각 스텝 FR의 활성화 조건 준수).
- FR-C5. 좌/우 스와이프 및 인디케이터 탭으로의 이동은 제공하지 않는다(입력 완료 후 "다음" 탭으로만 전진).

### 2.2 Step 1 — 성별
- FR-1.1. "성별을 선택해주세요" 타이틀과 "사주 해석의 기본 정보예요" 힌트를 표시한다.
- FR-1.2. 남/여 두 개의 큰 선택 박스를 좌우로 배치한다. 탭 시 해당 박스가 선택 상태(강조 테두리 + 볼드)로 바뀌고 다른 박스는 해제된다.
- FR-1.3. 하나를 선택하기 전까지 "다음" CTA는 비활성화된다.

### 2.3 Step 2 — 이름
- FR-2.1. "이름을 알려주세요" 타이틀과 "사주 리포트에 표시됩니다" 힌트를 표시한다.
- FR-2.2. 단일 텍스트 필드를 제공한다. 포커스 시 iOS 기본 키보드가 올라온다.
- FR-2.3. 필드 하단에 "* 본명이 아니어도 괜찮아요" 보조 문구를 표시한다.
- FR-2.4. 공백을 제외한 길이 1자 이상이면 "다음" CTA가 활성화된다. 최대 20자로 제한하되 초과 입력은 자연스럽게 잘라낸다.

### 2.4 Step 3 — 생년월일
- FR-3.1. "언제 태어나셨나요?" 타이틀과 "사주 분석의 가장 중요한 정보예요" 힌트를 표시한다.
- FR-3.2. 상단에 **양력 / 음력** 2버튼 세그먼트를 배치한다. 기본값은 **양력**.
- FR-3.3. 음력 선택 시 "윤달" 보조 체크박스를 함께 표시한다(해당 월에 윤달이 없으면 비활성).
- FR-3.4. 3-column 휠 피커(연 / 월 / 일)를 제공한다. 기본 포커스는 연 1990, 월 03, 일 15로 초기화한다(사용자 미입력 시 유효하지 않은 날짜 방지용).
- FR-3.5. 연도 선택 범위는 현재 연도 기준 **1900년 ~ 당해**로 제한한다.
- FR-3.6. 월/일 선택은 선택된 연월에 따라 존재하지 않는 날짜(예: 2월 30일)를 필터링한다.
- FR-3.7. 어떤 값이든 선택되어 있으면 "다음" CTA가 활성화된다(기본값 세팅으로 첫 진입 시에도 활성).

### 2.5 Step 4 — 태어난 시간
- FR-4.1. "몇 시에 태어나셨나요?" 타이틀과 "시주는 사주의 네 기둥 중 하나예요" 힌트를 표시한다.
- FR-4.2. 2-column 휠 피커(시 / 분)를 제공한다. 분 단위는 **15분 간격**(00/15/30/45)을 기본 옵션으로 제공하고 그 외 임의 값(예: 50분)도 휠에 포함한다.
- FR-4.3. 휠 피커 아래에 "시간을 모르겠어요 (시주 없이 3주 기반 분석, 나중에 업데이트 가능)" 체크박스를 표시한다.
- FR-4.4. "시간을 모르겠어요"가 체크되면 휠 피커가 **비활성화**(탭 불가, 흐린 색)되고, 입력 모델에 `hourKnown = false`를 기록한다.
- FR-4.5. "다음" CTA는 `hourKnown = true`(휠 선택)이거나 "모르겠어요" 체크 중 하나의 조건이 만족되면 활성화된다.

### 2.6 Step 5 — 출생지
- FR-5.1. "어디서 태어나셨나요?" 타이틀과 "진태양시 보정에 사용돼요" 힌트를 표시한다.
- FR-5.2. 돋보기 아이콘 + 텍스트 필드로 구성된 검색 박스를 제공한다. 포커스 시 iOS 키보드가 올라온다.
- FR-5.3. 검색 박스 아래에 **최근/기본 도시 리스트**(서울특별시, 부산광역시, 인천광역시, 대구광역시, 광주광역시, 대전광역시)를 제공한다. 첫 진입 시 서울특별시가 선택 상태.
- FR-5.4. 검색어 입력 시 국내 시/군/구 단위로 자동완성 리스트가 나타난다. 탭하면 선택 상태로 설정한다.
- FR-5.5. 하단에 "국외 출생 (경도 직접 입력)" 체크박스를 제공한다. 체크 시 도시 리스트 대신 **경도 직접 입력 필드**(°, 소수점 2자리까지)를 노출한다.
- FR-5.6. 도시가 선택되어 있거나 국외 경도 값이 유효한 범위(−180.00 ~ 180.00)에 있으면 "다음" CTA가 활성화된다.

### 2.7 Step 6 — 진태양시 보정
- FR-6.1. "진태양시로 보정할까요?" 타이틀과 "표준시와 실제 태양시의 차이를 보정해요" 힌트를 표시한다.
- FR-6.2. 상단에 "진태양시가 뭔가요? →" 밑줄 링크를 제공하고, 탭 시 **하단 시트(bottom sheet) 팝업**으로 설명(표준시 기준 경도, 예: 서울 −32분, 보정 효과)을 3개 카드 형태로 노출한다. 외부 영역 탭 또는 "확인" 버튼으로 닫는다.
- FR-6.3. "진태양시 보정" 토글 스위치를 제공한다. **기본값 ON**. 토글 옆 보조 문구: "정확한 분석을 위해 권장".
- FR-6.4. 토글 아래에 "계산 결과" 박스를 고정 표시: 출생지 경도(°), 표준시 차이(분), 보정된 출생시(시:분). 토글 OFF 시 차이·보정시는 "미적용"으로 바꾼다.
- FR-6.5. Step 4에서 `hourKnown = false`인 경우 Step 6은 **스킵**하고 바로 Step 7(로더)로 진행한다(진태양시는 시간 보정이므로 시주가 없으면 의미 없음).
- FR-6.6. "사주 분석 시작" CTA는 항상 활성화되어 있으며, 탭 시 입력 모델을 확정하고 Step 7로 진행한다.

### 2.8 Step 7 — 분석 로더
- FR-7.1. 중앙에 애니메이션 플레이스홀더, "사주를 분석 중입니다" 문구, 사용자 생년월일을 포함한 서브문구("{YYYY}년 {M}월 {D}일생의 원국을 계산하고 있어요")를 표시한다.
- FR-7.2. 프로그레스 바(0% → 100%)와 퍼센트 숫자를 표시한다. 실제 분석 완료 시점에 100%에 도달한다.
- FR-7.3. 로더 하단에 **명리학 팁** 카드를 표시하고, 팁 3개를 일정 간격(예: 2.5초)으로 페이드 교체한다. 하단 인디케이터 도트 3개로 현재 팁 위치를 표시한다.
- FR-7.4. 가장 아래에 "Swiss Ephemeris 기반 정밀 계산 중" 크레딧 문구를 표시한다.
- FR-7.5. 로더 표시 **최소 시간은 1.8초**이다. 실제 분석이 더 빨리 끝나도 최소 시간을 채워 Aha 모먼트의 기대감을 유지한다. 실제 분석이 길어지면 실제 시간만큼 표시한다.
- FR-7.6. 로더 중 뒤로 제스처는 무시하고, 완료 시 Step 8(결과)로 자동 전환한다.

### 2.9 Step 8 — 나의 투자 성향 (결과 단일 화면)
- FR-8.1. 화면 타이틀은 "나의 투자 성향"이며, 헤더 우측에 "다시 입력하기" 버튼이 있다. 탭 시 기존 결과를 비우고 Step 1 입력 화면으로 돌아간다.
- FR-8.2. 화면은 세로 스크롤 가능한 단일 화면이며, 아래 블록을 순서대로 포함한다.
  1. **Hero 유형 카드** — "{이름}님의 투자 성향" 레이블 + 유형 이름(예: "단단한 수집가형") + 일주/십성 요약(예: "경금(庚金) 일주 · 정재 중심") + 한 줄 설명.
  2. **사주 원국 미니 차트** — 4주(시/일/월/년) 천간·지지 셀, 일간 강조, 입력·보정 메타(양력 {날짜} {시간}, 출생지, 진태양시 보정 여부). 우측에 "내 사주 자세히 보기 →" 링크.
  3. **오행(五行) 균형 바 차트** — 5개 원소(목·화·토·금·수) 막대 + 경고 문구(예: "水 부재 · 火·金 과다 — 충동 판단 제어 필요").
  4. **강점** — 불릿 3개(예: 분석력·인내심, 큰 흐름 파악, 리스크 관리).
  5. **주의점** — 빨간 불릿 3개(예: 충동 매수 주의, 수익 실현 타이밍, 과도한 분석).
  6. **성향에 맞는 접근 참고** — 불릿 3개(예: 분산 투자, 중기 흐름, 체크리스트 루틴).
  7. **입력 정보 요약** — "{생년월일} {시간} · {출생지} · {성별}" + "수정" 버튼(탭 시 해당 스텝으로 이동).
- FR-8.3. Hero 카드 상단 라벨의 "{이름}님" 부분은 이름 길이가 10자 초과일 경우 8자 + "…"로 말줄임 처리한다. 이름이 비어있으면 "당신의 투자 성향"으로 대체한다.
- FR-8.4. **정확도 배지**: 시간·출생지·진태양시 모두 입력/보정 → "정확도: 높음". 시간만 입력 → "정확도: 보통". 시간 미입력 → "정확도: 보통" + "시간 추가하면 정교해져요 →" CTA(탭 시 Step 4로 이동).
- FR-8.5. `hourKnown = false`인 경우 사주 원국 미니 차트의 시주(時柱) 칼럼은 **빈 대시 + 대시 테두리**로 렌더되며, "미입력" 라벨을 표시한다.
- FR-8.6. 하단에 **"시작하기"(primary)**, **"공유하기"(secondary)** 버튼과 법적 고지(Disclaimer) 문구를 배치한다.
- FR-8.7. "수정" 탭 시 해당 스텝의 입력값을 유지한 상태로 진입하고, 변경 후 완료 시 결과 화면으로 돌아오며 재분석이 자동 수행된다.
- FR-8.8. "공유하기" 탭 시 공유 카드를 생성하여 iOS 네이티브 공유 시트(UIActivityViewController)를 띄운다. (카드 디자인은 Step 10 Referral 섹션과 동일 컴포넌트 재사용.)

### 2.10 Step 8.5 — 회원가입 (결과 후 유도)
- FR-8.5.1. Step 8에서 "시작하기" CTA를 탭하면 회원가입 화면으로 이동한다. 단, 이미 로그인된 세션이면 이 화면을 스킵하고 홈(WF3)으로 이동한다.
- FR-8.5.2. 화면 타이틀: "결과 저장하기". 상단에 "방금 본 결과를 계정에 저장해두면 언제든 다시 볼 수 있어요" 설명 문구.
- FR-8.5.3. Apple / Google / 이메일 3개 가입 버튼을 제공한다(실제 인증 연동은 별도 스펙).
- FR-8.5.4. 하단에 "나중에 하기" 링크를 제공한다. 탭 시 비회원 상태로 홈(WF3)으로 이동하되, 분석 결과는 이 세션 동안 로컬에만 유지한다.
- FR-8.5.5. 법적 고지: "가입 시 서비스 이용약관 및 개인정보 처리방침에 동의합니다." 링크 2개 제공.

### 2.11 Step 10 — 친구 초대 + 공유 카드
- FR-10.1. 홈(WF3)에서 특정 진입점(배너/세팅)을 통해 진입하는 독립 화면이며, WF2 흐름 종료 직후에는 자동 진입하지 않는다.
- FR-10.2. 화면 타이틀: "친구 초대". 헤더 우측에 "내 코드: {5자리 영숫자}" 표시.
- FR-10.3. 상단에 "친구 초대 혜택" 카드: "나 — 가입 완료 쿠폰 5,000원", "친구 — 초대 보상 1,000P".
- FR-10.4. 중앙에 **공유 카드 미리보기**(ShareCard 컴포넌트): 유형명, 일주/십성, 오행 미니바, 한 줄 설명, 하단 "운테크.app" + 오늘 날짜.
- FR-10.5. 공유 CTA 3개: "인스타그램 스토리로 공유"(primary), "링크 복사하기", "카카오톡으로 공유".
- FR-10.6. 하단 "내 초대 코드" 카드: 5자리 코드 + 복사 버튼 + "초대한 친구: {N}명 · 받은 보상: {M}P" 요약.
- FR-10.7. 인스타그램 공유는 iOS Instagram Stories URL scheme을 사용하여 배경 이미지로 공유 카드를, 스티커 레이어로 초대 코드를 전달한다. 인스타그램 미설치 시 앱스토어 링크로 대체한다.
- FR-10.8. 카카오톡 공유는 Kakao Link SDK 사용(별도 연동). 링크 복사는 iOS Pasteboard에 초대 URL(`https://woontech.app/invite/{CODE}`)을 기록하고 토스트 "복사되었어요"를 노출한다.

## 3. Non-functional Constraints

- NFC-1. **플랫폼**: iOS 네이티브(SwiftUI 권장). 세이프 에어리어를 준수하고 키보드 노출 시 입력 필드가 가려지지 않도록 스크롤을 자동 조정한다.
- NFC-2. **성능**: 사주 계산(진태양시 보정 · 4주 · 십성 · 오행)은 디바이스에서 200ms 이내에 완료되어야 하며, 로더는 FR-7.5의 최소 1.8초 규칙을 준수한다. 스텝 간 전환 애니메이션은 300ms 이내.
- NFC-3. **정확도**: 진태양시 보정은 Swiss Ephemeris 기반 균시차·경도 보정을 적용한다. 절입시(예: 경칩)에 따른 월주 경계 처리는 `algorithm_of_saju.md` 규칙을 따른다.
- NFC-4. **접근성**: 각 스텝의 입력 요소, CTA, 체크박스, 토글, 휠 피커는 VoiceOver 레이블과 현재 값을 트레잇으로 노출한다. 결과 화면의 사주 원국 셀은 "{천간}{지지} · {오행}"로 읽힌다.
- NFC-5. **터치 타겟**: 뒤로 버튼, 토글, 체크박스, "수정" 버튼의 hit target은 44×44pt 이상을 만족한다.
- NFC-6. **데이터 영속성**: Step 1~6 입력값은 스텝 이동 시 메모리에 임시 유지되고, Step 8 결과가 렌더될 때 로컬 영속 저장소(UserDefaults 또는 Core Data)에 `userProfile`로 저장된다. Step 8.5에서 가입을 완료하면 서버로 업로드된다.
- NFC-7. **다국어**: 모든 텍스트 리터럴은 로컬라이즈 키로 분리한다. 1차 릴리스는 한국어만 제공.
- NFC-8. **법적 고지**: 결과 화면 하단(FR-8.6)의 Disclaimer는 "본 앱은 학습·참고용이며 투자 권유가 아닙니다" 문구를 반드시 포함한다.
- NFC-9. **공유 카드 이미지 생성**: SwiftUI 뷰를 UIImage로 렌더링(`ImageRenderer`)하여 공유 시트에 전달한다. 1080×1920 해상도 고정(인스타 스토리 규격).

## 4. Out of Scope

- **회원가입 실제 인증 연동**(Apple / Google / 이메일 / Kakao OAuth) — 별도 인증 스펙에서 정의. 본 스펙은 UI 및 네비게이션만 다룬다.
- **홈(WF3)·사주 탭(WF4)·투자 탭(WF5)·마이(WF6)** 및 그 하위의 모든 상세 화면 — 별도 WF 범위.
- **PRO 페이월 결제 플로우** — 결제·구독 관리 스펙에서 별도 정의. 본 스펙에서 결과 화면 내 페이월 컴포넌트 자체는 표시하되 결제 진입 동작은 "페이월 진입점으로 이동" 수준까지만 정의.
- **A/B 테스트, 애널리틱스 이벤트 정의** — 추후 분석 스펙.
- **대운(大運) 타임라인, 상세 성향 리포트** 등 PRO 콘텐츠 — 결과 화면에서는 잠금 상태(locked)로만 노출.
- **오늘의 일진, 십성 분포, 합충 상세**(ScrTodayDetailV2 / 사주 탭에 속함) — WF4 범위.
- **즐겨찾기 인물 사주**(배우자/어머니/자녀) — WF3 홈 범위.
- **다크 모드, 영어 로컬라이제이션, 아이패드 레이아웃** — 별도 이슈.
- **Kakao Link SDK / Instagram URL Scheme 실제 공유** 구현 — 본 스펙은 CTA와 표시되는 내용까지만 정의. 실제 연동은 공유 기능 스펙에서 처리.
- **시 단위 이하 정확한 출생지 좌표 매핑 DB** 구축 — 1차에는 시/도 단위 고정 경도 테이블을 사용한다.

## 5. Acceptance Criteria

각 항목은 UI 테스트 또는 유닛 테스트로 검증 가능해야 한다.

1. 온보딩 완료 후 Step 1(성별)으로 진입하고, 상단 진행 바가 1/6로 표시된다.
2. Step 1에서 남 또는 여를 선택하기 전까지 "다음" CTA는 비활성화 상태이다. 선택하면 즉시 활성화된다.
3. Step 2에서 이름 필드가 비어 있으면 "다음" CTA가 비활성, 1자 이상 입력하면 활성화된다. 20자 초과 입력은 자동으로 20자까지만 허용된다.
4. Step 3 최초 진입 시 연/월/일 휠의 기본값이 각각 1990, 03, 15로 세팅되고 "다음" CTA가 활성 상태이다. 양력/음력 토글이 기본 양력으로 선택되어 있다.
5. Step 3에서 음력 선택 시 "윤달" 체크박스가 노출된다. 연도 범위는 1900 ~ 당해 연도로 제한된다. 존재하지 않는 날짜(2월 30일 등)는 휠에서 선택 불가능하다.
6. Step 4에서 휠로 시/분을 선택하면 "다음" CTA가 활성화된다.
7. Step 4에서 "시간을 모르겠어요" 체크 시 휠 피커가 비활성화되고 "다음" CTA가 활성화된다. 입력 모델에 `hourKnown = false`가 기록된다.
8. Step 5 최초 진입 시 서울특별시가 선택되어 있으며 "다음" CTA는 활성 상태이다.
9. Step 5 "국외 출생" 체크 시 도시 리스트가 숨겨지고 경도 입력 필드가 나타나며, -180.00 ~ 180.00 범위의 값이 입력되어야 "다음"이 활성화된다.
10. Step 4에서 `hourKnown = false`였으면 Step 5 "다음" 탭 시 Step 6을 **스킵**하고 Step 7 로더로 바로 진입한다.
11. Step 6 진입 시 진태양시 토글이 기본 ON이고, "계산 결과" 박스에 경도·표준시 차이(−32분 등)·보정된 출생시가 표시된다. 토글 OFF 시 차이·보정시가 "미적용"으로 바뀐다.
12. Step 6의 "진태양시가 뭔가요?" 링크 탭 시 하단 시트 팝업이 나타나며, 외부 영역 탭 또는 "확인" 버튼으로 닫힌다.
13. Step 6 "사주 분석 시작" CTA 탭 시 Step 7 로더로 전환된다.
14. 로더 화면에는 프로그레스 바, 퍼센트, 명리학 팁(일정 간격으로 3개 교체), "Swiss Ephemeris 기반 정밀 계산 중" 문구가 표시된다. 표시 시간은 최소 1.8초를 보장한다.
15. Step 8 결과 화면은 단일 스크롤 화면으로 Hero → 사주 원국 → 오행 → 강점 → 주의점 → 접근 참고 → 입력 정보 순서로 렌더된다.
16. Step 8 Hero 카드 라벨에 사용자의 이름이 "{이름}님의 투자 성향" 포맷으로 표시된다. 이름이 10자 초과면 8자 + "…"로 말줄임되고, 비어 있으면 "당신의 투자 성향"으로 대체된다.
17. Step 8 정확도 배지는 FR-8.4 규칙에 따라 "높음 / 보통 / 보통 + 시간 추가 CTA" 중 하나로 표시된다.
18. `hourKnown = false`인 경우 사주 원국 미니 차트의 시주 칼럼이 대시 테두리 + "미입력"으로 렌더된다.
19. Step 8 "수정" 탭 시 해당 스텝으로 이동하고, 입력 변경 완료 시 결과 화면으로 복귀하며 재분석이 자동 반영된다(예: 이름 변경 → Hero 라벨 반영).
20. Step 8 "공유하기" 탭 시 1080×1920 공유 카드 이미지가 iOS 공유 시트에 첨부되어 나타난다.
21. Step 8 "시작하기" 탭 시 Step 8.5 회원가입 화면으로 이동한다(이미 로그인된 세션은 홈으로 바로 이동).
22. Step 8.5 "나중에 하기" 탭 시 비회원 상태로 홈으로 이동하고, 분석 결과는 세션 내 로컬에 유지된다.
23. Step 10 친구 초대 화면은 홈의 진입점(배너/세팅)을 통해서만 진입하며, WF2 플로우 종료 직후에는 자동 진입하지 않는다.
24. Step 10에 표시되는 공유 카드 미리보기, "내 코드" 표시, 초대 혜택 카드의 값이 현재 사용자 프로필과 일치한다.
25. Step 10 "링크 복사하기" 탭 시 `https://woontech.app/invite/{CODE}` 형식이 pasteboard에 복사되고 "복사되었어요" 토스트가 표시된다.
26. 모든 입력 요소(필드, 체크박스, 토글, 휠 피커, CTA)는 VoiceOver로 접근 가능하며 현재 상태(선택/체크/값)가 트레잇으로 노출된다.
27. 뒤로 버튼, 토글, 체크박스, "수정" 버튼의 hit target은 44×44pt 이상을 만족한다(accessibility inspector 또는 hit-test 검증).
28. Step 8 결과 화면 하단에 "본 앱은 학습·참고용이며 투자 권유가 아닙니다" 문구가 포함된 Disclaimer가 항상 표시된다.
